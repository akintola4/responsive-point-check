# Code of Conduct

This project is open to anyone willing to engage constructively. The
expectations below apply to everyone — contributors, maintainers, and
anyone using the issue tracker or pull requests.

## Be a good colleague

- Treat people the way you'd want a co-worker to treat you.
- Critique code, not people. Disagreements happen — keep them about the
  technical question on the table.
- English is a second language for many contributors. Read charitably.

## Not okay

- Personal attacks, name-calling, or sustained disruption.
- Posting someone's private information without permission.
- Off-topic content that derails issues or PR threads.

## Reporting

If something here goes wrong, email **akintope830@gmail.com**. Reports
are read by the maintainer and stay private. Outcomes can range from a
warning, to having a comment edited or removed, to being blocked from
the repository.

## Scope

This applies in every space the project owns — issues, pull requests,
discussions, and any project-affiliated communication. Acting on behalf
of the project elsewhere (e.g. representing RPC at an event) is in
scope too.
